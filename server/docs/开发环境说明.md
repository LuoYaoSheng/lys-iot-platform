# 开发模式 vs 生产模式说明

作者：罗耀生
日期：2025-12-16

## 📋 两种部署模式

### 1️⃣ **开发模式**（推荐日常开发使用）

**特点：**
- ✅ 只启动基础服务（MySQL、Redis、EMQX）
- ✅ 后端手动运行，方便调试和热重载
- ✅ 使用 `48080 / 48306 / 48379 / 48883 / 48884` 这一组本地开发端口

**启动方式：**
```bash
# 1. 启动基础服务
make dev-infra-up

# 2. 手动运行后端（支持断点调试）
go run ./cmd/server

# 或使用 IDE 调试功能
```

**适用场景：**
- 日常开发调试
- 修改后端代码后立即生效
- 使用IDE断点调试
- 前端联调测试

---

### 2️⃣ **生产模式**（用于教程演示/正式部署）

**特点：**
- ✅ 一键启动所有服务
- ✅ 后端容器化运行，稳定可靠
- ✅ 适合录制视频、演示教程

**启动方式：**
```bash
# 一键启动所有服务（包括后端）
docker compose up -d
```

**适用场景：**
- 录制教程视频
- 生产环境部署
- 演示给他人使用
- 长期稳定运行

---

## 🔄 切换模式

### 从开发模式切换到生产模式：

```bash
# 1. 停止开发环境
docker compose -f docker-compose.dev.yml down

# 2. 停止手动运行的后端（Ctrl+C）

# 3. 启动生产环境
docker compose up -d
```

### 从生产模式切换到开发模式：

```bash
# 1. 停止生产环境
docker compose down

# 2. 启动开发环境
make dev-infra-up

# 3. 手动运行后端
go run ./cmd/server
```

## ⚠️ 注意事项

### 端口冲突检查

如果遇到端口冲突，检查是否有多个实例在运行：

```bash
# 检查Docker服务状态
docker compose ps

# 检查关键端口占用
netstat -ano | findstr "48080"
netstat -ano | findstr "48306"
netstat -ano | findstr "48379"
netstat -ano | findstr "48883"
netstat -ano | findstr "48884"

# Windows：结束进程
taskkill /F /PID <进程ID>
```

### 数据持久化

两种模式**共享同一个数据库**（Docker volume），切换模式不会丢失数据。

### EMQX认证配置差异

| 配置项 | 开发模式 | 生产模式 |
|--------|---------|---------|
| 后端地址 | `host.docker.internal:48080` | `iot-platform-core:48080` |
| 说明 | 连接宿主机的后端服务 | 连接容器内的后端服务 |

## 🎯 推荐工作流程

**日常开发：**
```bash
# 早上启动开发环境
make dev-infra-up
go run ./cmd/server

# 晚上关闭
make dev-infra-down
```

**录制教程前：**
```bash
# 切换到生产模式
docker compose down
make dev-infra-down
docker compose up -d

# 演示完毕后切回开发模式
docker compose down
make dev-infra-up
```

## 📝 文件说明

| 文件 | 用途 | 后端服务 |
|------|------|---------|
| `docker-compose.dev.yml` | 开发模式 | ❌ 不包含（手动运行） |
| `docker-compose.yml` | 生产模式 | ✅ 包含（容器运行） |

---

## 💡 快捷命令

可以在 `.bashrc` 或 `.zshrc` 中添加别名：

```bash
# 开发环境
alias dev-start="make dev-infra-up && go run ./cmd/server"
alias dev-stop="make dev-infra-down"

# 生产环境
alias prod-start="docker compose up -d"
alias prod-stop="docker compose down"
```

## ❓ FAQ

**Q: 我忘记用哪个模式了，怎么检查？**

A: 运行 `docker compose ps`，如果看到 `iot-platform_core` 容器，说明是生产模式。

**Q: 两种模式可以同时运行吗？**

A: ❌ 不可以，会导致48080端口冲突。

**Q: 数据会不会因为切换模式而丢失？**

A: ✅ 不会，数据存储在Docker volume中，两种模式共享。

**Q: 开发时修改代码后需要重新构建Docker镜像吗？**

A: ❌ 不需要，使用开发模式（`docker-compose.dev.yml`），手动运行后端即可。

**Q: 如何更新生产模式的后端镜像？**

A:
```bash
docker compose pull
docker compose up -d
```
