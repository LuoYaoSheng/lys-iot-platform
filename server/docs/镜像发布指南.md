# Docker 镜像发布指南

> 让团队成员可以直接拉取你的镜像，而不需要编译代码

---

## 🎯 一次性配置（首次使用）

### 1. 注册 Docker Hub 账号

访问: https://hub.docker.com/
- 注册账号（假设用户名是: `luoyaosheng`）
- 记住你的用户名和密码

### 2. 登录 Docker Hub

```bash
docker login
# 输入你的用户名和密码
```

---

## 📦 发布镜像（每次更新代码后）

### 方式一：自动构建并发布（推荐）

创建一个发布脚本 `publish.sh`：

```bash
#!/bin/bash
# 自动构建并发布到 Docker Hub

# 1. 构建镜像
echo "🔨 正在构建镜像..."
docker build -t luoyaosheng/iot-platform-core:latest .

# 2. 打标签（版本号）
VERSION=$(date +%Y%m%d-%H%M%S)
docker tag luoyaosheng/iot-platform-core:latest luoyaosheng/iot-platform-core:$VERSION

# 3. 推送到 Docker Hub
echo "📤 正在推送镜像..."
docker push luoyaosheng/iot-platform-core:latest
docker push luoyaosheng/iot-platform-core:$VERSION

echo "✅ 发布完成！"
echo "   最新版本: luoyaosheng/iot-platform-core:latest"
echo "   带版本号: luoyaosheng/iot-platform-core:$VERSION"
```

然后运行：

```bash
chmod +x publish.sh
bash publish.sh
```

### 方式二：手动构建并发布

```bash
# 1. 构建镜像
docker build -t luoyaosheng/iot-platform-core:latest .

# 2. 推送到 Docker Hub
docker push luoyaosheng/iot-platform-core:latest
```

## 🔐 私有镜像（可选）

如果不想公开镜像，可以使用：

### 选项1: Docker Hub 私有仓库（收费）
- Docker Hub 提供 1 个免费私有仓库
- 更多私有仓库需要付费

### 选项2: 阿里云镜像仓库（免费）

1. 访问: https://cr.console.aliyun.com/
2. 创建命名空间和仓库
3. 按照阿里云的指引推送镜像

```bash
# 登录阿里云
docker login --username=你的阿里云账号 registry.cn-hangzhou.aliyuncs.com

# 打标签
docker tag luoyaosheng/iot-platform-core:latest registry.cn-hangzhou.aliyuncs.com/你的命名空间/iot-platform-core:latest

# 推送
docker push registry.cn-hangzhou.aliyuncs.com/你的命名空间/iot-platform-core:latest
```

### 选项3: 自建 Docker Registry（最灵活）

在宝塔服务器上搭建私有仓库：

```bash
# 启动私有仓库
docker run -d -p 5000:5000 --name registry registry:2

# 推送镜像到私有仓库
docker tag luoyaosheng/iot-platform-core:latest localhost:5000/iot-platform-core:latest
docker push localhost:5000/iot-platform-core:latest
```

## 📋 更新镜像流程

每次代码更新后：

```bash
# 1. 提交代码
git add .
git commit -m "更新代码"
git push

# 2. 发布新镜像
bash publish.sh

# 3. 通知团队成员更新
# 团队成员只需要运行:
docker pull luoyaosheng/iot-platform-core:latest
docker compose -f docker-compose.simple.yml up -d --force-recreate
```
