## 密码安全

### 1. 生成强密码
```bash
# 生成随机密码（16位）
openssl rand -base64 16 | tr -d "=+/" | cut -c1-16

# 生成随机密码（32位）
openssl rand -base64 32

# 生成 UUID
uuidgen
```

### 2. 必须修改的默认密码

#### MySQL root 密码
```bash
# 进入 MySQL 容器
docker exec -it iot-platform_mysql mysql -uroot -proot123456

# 修改密码
ALTER USER 'root'@'%' IDENTIFIED BY '新的强密码';
ALTER USER 'root'@'localhost' IDENTIFIED BY '新的强密码';
FLUSH PRIVILEGES;
EXIT;
```

#### MySQL 业务用户密码
```bash
# 修改 .env 文件中的 DB_PASSWORD
DB_PASSWORD=新生成的强密码

# 在 MySQL 中同步修改
docker exec -it iot-platform_mysql mysql -uroot -p
ALTER USER 'iot_user'@'%' IDENTIFIED BY '新生成的强密码';
FLUSH PRIVILEGES;

# 重启服务使配置生效
docker compose restart iot-platform-core
```

#### EMQX Dashboard 密码
```bash
# 访问 http://your_server_ip:48884
# 登录: admin / public
# 路径: 系统 -> 用户 -> 修改密码
```

#### JWT 密钥
```bash
# 生成新的 JWT 密钥
JWT_SECRET=$(openssl rand -base64 32)

# 更新 .env 文件
echo "JWT_SECRET=$JWT_SECRET" >> .env

# 重启服务
docker compose restart iot-platform-core
```

## 防火墙配置

### 基本原则
- 只开放必要的端口
- 限制管理端口的访问IP
- 禁用不必要的服务

### Ubuntu (UFW)
```bash
# 重置防火墙（可选）
sudo ufw --force reset

# 默认策略
sudo ufw default deny incoming
sudo ufw default allow outgoing

# SSH（必须保留，否则会被锁在外面）
sudo ufw allow 22/tcp

# API 服务（公开访问）
sudo ufw allow 48080/tcp

# MQTT（公开访问）
sudo ufw allow 48883/tcp

# EMQX Dashboard（限制IP访问）
sudo ufw allow from 你的IP地址 to any port 48884

# 启用防火墙
sudo ufw enable

# 查看规则
sudo ufw status numbered
```

### CentOS (firewalld)
```bash
# 添加规则
sudo firewall-cmd --permanent --add-port=22/tcp
sudo firewall-cmd --permanent --add-port=48080/tcp
sudo firewall-cmd --permanent --add-port=48883/tcp

# 限制 EMQX Dashboard 访问
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="你的IP地址" port protocol="tcp" port="48884" accept'

# 重载配置
sudo firewall-cmd --reload

# 查看规则
sudo firewall-cmd --list-all
```

### 云服务商安全组
在云控制台配置：
- **入站规则**:
  - TCP 22 (SSH) - 仅允许特定IP
  - TCP 48080 (API) - 0.0.0.0/0
  - TCP 48883 (MQTT) - 0.0.0.0/0
  - TCP 48884 (EMQX Dashboard) - 仅允许特定IP
- **出站规则**: 允许所有

## SSL/TLS 配置

### 使用 Let's Encrypt 免费证书

#### 1. 安装 Certbot
```bash
sudo apt update
sudo apt install certbot -y
```

#### 2. 获取证书
```bash
# 停止占用80端口的服务
docker compose stop

# 获取证书
sudo certbot certonly --standalone -d your-domain.com

# 证书位置: /etc/letsencrypt/live/your-domain.com/
```

#### 3. 配置 Nginx 反向代理
```bash
# 安装 Nginx
sudo apt install nginx -y

# 创建配置文件
sudo nano /etc/nginx/sites-available/iot-platform
```

配置内容:
```nginx
# API 服务
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        proxy_pass http://localhost:48080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

启用配置:
```bash
sudo ln -s /etc/nginx/sites-available/iot-platform /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

#### 4. 自动续期证书
```bash
# 测试续期
sudo certbot renew --dry-run

# 添加定时任务
sudo crontab -e
# 每月1号凌晨3点尝试续期
0 3 1 * * certbot renew --quiet && systemctl reload nginx
```

### MQTT over TLS (MQTTS)

修改 `docker-compose.yml`:
```yaml
emqx:
  ports:
    - "8883:8883"  # MQTTS 端口
  volumes:
    - /etc/letsencrypt:/etc/letsencrypt:ro
```

在 EMQX Dashboard 配置 TLS:
1. 访问 Dashboard
2. 路径: 管理 -> 监听器 -> SSL
3. 添加证书路径

## EMQX 安全加固

### 1. 禁用匿名访问
```bash
# 在 docker-compose.yml 添加
environment:
  EMQX_ALLOW_ANONYMOUS: "false"

# 重启
docker compose restart emqx
```

### 2. 配置 ACL（访问控制列表）
在 EMQX Dashboard:
1. 路径: 访问控制 -> 认证
2. 配置设备只能发布/订阅自己的主题
3. 示例规则:
   ```
   允许: {username}/data/up
   允许: {username}/cmd/down
   禁止: #
   ```

### 3. 限流配置
```yaml
# docker-compose.yml
environment:
  EMQX_MQTT__MAX_PACKET_SIZE: 1MB
  EMQX_ZONE__EXTERNAL__RATE_LIMIT: "1000,1s"
```

### 4. 定期审计认证日志
```bash
# 查看认证失败的记录
docker compose logs emqx | grep "authentication_failure"

# 查看数据库中的认证日志
docker exec -it iot-platform_mysql mysql -uiot_user -p
SELECT * FROM mqtt_auth_logs WHERE result = 'deny' ORDER BY created_at DESC LIMIT 100;
```

## 数据库安全

### 1. 限制远程访问
默认配置只允许容器内访问，不暴露到外网。
如需远程管理，使用 SSH 端口转发:
```bash
ssh -L 48306:localhost:48306 user@your_server_ip
```

### 2. 定期备份
```bash
# 手动备份
./manage.sh  # 选择 7. 数据库备份

# 自动备份（每天凌晨3点）
crontab -e
0 3 * * * cd /opt/iot-platform && ./manage.sh backup_database
```

### 3. 审计日志
```sql
-- 启用审计日志
SET GLOBAL general_log = 'ON';
SET GLOBAL general_log_file = '/var/log/mysql/general.log';

-- 查看慢查询
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```

## 监控告警

### 基础监控脚本
```bash
#!/bin/bash
# /opt/iot-platform/monitor.sh

# 检查服务状态
if ! docker compose ps | grep -q "Up (healthy)"; then
    echo "警告: 服务异常" | mail -s "IoT Platform Alert" your@email.com
fi

# 检查磁盘空间
DISK_USAGE=$(df -h / | tail -1 | awk '{print $5}' | sed 's/%//')
if [ $DISK_USAGE -gt 80 ]; then
    echo "警告: 磁盘使用率 ${DISK_USAGE}%" | mail -s "Disk Alert" your@email.com
fi

# 检查 API 健康
if ! curl -f -s http://localhost:48080/health > /dev/null; then
    echo "警告: API 服务异常" | mail -s "API Alert" your@email.com
fi
```

添加定时任务:
```bash
crontab -e
*/5 * * * * /opt/iot-platform/monitor.sh
```

## 应急响应

### 发现异常连接
```bash
# 查看当前连接
netstat -tnp | grep -E "48080|48883"

# 查看 EMQX 客户端
docker exec -it iot-platform_emqx emqx ctl clients list

# 踢出异常客户端
docker exec -it iot-platform_emqx emqx ctl clients kick <client_id>
```

### 数据泄露应急
1. 立即修改所有密码
2. 重新生成 JWT 密钥
3. 查看访问日志，确定泄露范围
4. 通知受影响用户
5. 更新安全策略
