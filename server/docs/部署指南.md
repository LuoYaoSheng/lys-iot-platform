# IoT 平台 - 极简部署指南

> **目标**: 让团队成员在宝塔环境下，4步完成部署

---

## 🎯 部署步骤（超简单！）

### 1️⃣ 上传文件到服务器

把这 3 个文件上传到宝塔服务器：
- `docker-compose.simple.yml`
- `.env.simple`
- `deploy.sh`

或者直接拉取整个项目：
```bash
cd /www/wwwroot
git clone https://gitee.com/你的用户名/iot-platform-core.git
cd iot-platform-core
```

### 2️⃣ 修改配置（只需要改1个地方！）

```bash
# 复制配置模板
cp .env.simple .env

# 编辑配置文件
vi .env
```

**只需要修改这一行：**
```bash
SERVER_IP=192.168.1.100  # 改成你的服务器IP或域名
```

其他配置可以保持默认，或者按需修改：
```bash
DB_PASSWORD=root123456   # 数据库密码（可选）
JWT_SECRET=随机字符串    # JWT密钥（可选）
```

### 3️⃣ 一键部署

```bash
bash deploy.sh
```

等待 30 秒，部署完成！

### 🎉 自动初始化

**最新版本（v20251219+）已内置数据库自动初始化功能：**
- ✅ 应用启动时自动检查产品表是否为空
- ✅ 如果为空，自动插入默认产品数据（SW-SERVO-001）
- ✅ **无需手动执行 SQL 脚本**
- ✅ **无需担心 `invalid_product_key` 错误**

查看应用日志确认初始化成功：
```bash
docker compose -f docker-compose.simple.yml logs -f iot-core | grep -E "(Checking default data|Created product)"
```

### 4️⃣ 配置 EMQX（重要！必须完成）

部署完成后，**必须**配置 EMQX 才能正常工作。EMQX 需要配置 3 项：

#### ⚠️ HTTP 认证（必须手动配置）
**重要**：虽然 docker-compose 配置了环境变量，但 EMQX 5.3.0 环境变量可能不会自动生效，**必须手动配置**。

#### ⚠️ HTTP 授权/ACL（必须手动配置）
控制设备只能访问自己的主题，防止越权访问。

#### ⚠️ Webhook（必须手动配置）
监听设备上下线事件，实时同步设备状态。

---

### 快速配置步骤

登录 EMQX：`http://你的IP:49084`（账号: `admin` / 密码: `public`）

#### 步骤 0: 配置 HTTP 认证（必须先配置！）

1. 左边菜单：**访问控制** → **客户端认证** → 点击 **+ 创建**
2. 选择类型：**HTTP Server**
3. 填写配置：
   ```
   名称: http_authn
   启用: ✅ 勾选

   HTTP 配置:
   方法: POST
   URL: http://iot-core:48080/api/v1/mqtt/auth

   请求头:
   Content-Type: application/json

   请求体:
   {
     "username": "${username}",
     "password": "${password}",
     "clientid": "${clientid}"
   }
   ```
4. 点击 **创建** 保存

#### 步骤 1: 配置 ACL 授权

1. 左边菜单：**访问控制** → **客户端授权** → 点击 **+ 创建**
2. 选择类型：**HTTP Server**
3. 填写配置：
   ```
   名称: http_acl
   启用: ✅ 勾选

   HTTP 配置:
   URL: http://iot_core:48080/api/v1/mqtt/acl
   方法: POST

   请求头:
   Content-Type: application/json

   请求体模板:
   {
     "username": "${username}",
     "topic": "${topic}",
     "action": "${action}"
   }
   ```
4. 点击 **创建** 保存

#### 步骤 2: 配置设备上线 Webhook

1. 左边菜单：**集成** → **Webhook** → 点击 **+ 创建**
2. 填写配置：
   ```
   名称: device_online
   启用: ✅ 勾选

   触发事件:
   选择: client.connected（客户端连接）

   动作配置:
   URL: http://iot_core:48080/api/v1/webhook/client
   方法: POST
   ```
3. 点击 **创建** 保存

#### 步骤 3: 配置设备下线 Webhook

1. 再次点击 **+ 创建**
2. 填写配置：
   ```
   名称: device_offline
   启用: ✅ 勾选

   触发事件:
   选择: client.disconnected（客户端断开连接）

   动作配置:
   URL: http://iot_core:48080/api/v1/webhook/client
   方法: POST
   ```
3. 点击 **创建** 保存

---

### 验证配置

检查配置是否成功：

- **访问控制** → **客户端认证** 页面应该有：
  - ✅ `http_authn` - 已启用 - HTTP Server

- **访问控制** → **客户端授权** 页面应该有：
  - ✅ `http_acl` - 已启用

- **集成** → **Webhook** 页面应该有：
  - ✅ `device_online` - 已启用
  - ✅ `device_offline` - 已启用

**📘 详细配置说明**: 参考 [EMQX 完整配置指南](./EMQX完整配置指南.md)

## 📚 公共库使用

### Go 项目引用公共库

在你的 Go 项目中：

```bash
# 1. 引用公共库（如果是 Gitee）
go get gitee.com/你的用户名/iot-libs-common/go-sdk

# 2. 在代码中导入
import "gitee.com/你的用户名/iot-libs-common/go-sdk/mqtt"
```

### Flutter 项目引用公共库

在 `pubspec.yaml` 中：

```yaml
dependencies:
  iot_sdk:
    git:
      url: https://gitee.com/你的用户名/iot-libs-common.git
      path: flutter-sdk
      ref: main
```

## ❓ 常见问题

### 1. 端口被占用？

如果端口冲突，修改 `docker-compose.simple.yml` 中的端口映射：

```yaml
ports:
  - "48080:48080"  # 改成 "你想要的端口:48080"
```

### 2. 镜像拉取失败？

可能是网络问题，手动拉取：

```bash
docker pull luoyaosheng/iot-platform-core:latest
```

或者使用阿里云镜像加速器（在宝塔Docker管理中配置）。

### 3. 服务启动失败？

查看日志：

```bash
docker compose -f docker-compose.simple.yml logs
```
